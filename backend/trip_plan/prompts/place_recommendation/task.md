根据用户输入的一段文化旅行需求，推荐 3 个左右最匹配的地市。

你需要分析：
- 文化主题
- 旅行人群
- 时间长度
- 预算倾向
- 兴趣偏好
- 体验方式

输出字段必须包含：
- id：前端静态数据中的目的地 id，例如 xian、hangzhou、beijing、nanjing、chengdu、dunhuang
- province：省份、直辖市或自治区
- city：地市名称
- matchScore：0-100 的整数匹配度
- tags：适合前端展示的标签数组
- reasons：推荐理由数组
- intro：目的地简介

如果用户文本信息不足，也请给出适合文化旅行的通用推荐。
如果用户偏好明显，例如唐代文化、江南、博物馆、非遗、亲子研学、丝路文化，应优先推荐最匹配的地市。
不要输出完整行程规划。
不要编造实时门票、开放时间。
